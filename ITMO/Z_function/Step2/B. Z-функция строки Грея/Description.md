# B. Z-функция строки Грея

- **Ограничение по времени на тест:** 2 секунды
- **Ограничение по памяти на тест:** 512 мегабайт

Эта задача не требует знания эффективного алгоритма нахождения z-функции. Достаточно только понимания определения, чтобы
решить эту задачу.

Последовательность строк Грея *g*<sub>1</sub>, *g*<sub>2</sub>, … выглядит следующим образом:

- *g*<sub>1</sub> = «a»
- *g*<sub>2</sub> = «aba»
- *g*<sub>3</sub> = «abacaba»
- *g*<sub>4</sub> = «abacabadabacaba»
- ...
- *g*<sub>*i*</sub> = *g*<sub>*i*−1</sub> + *c*<sub>*i*</sub> + *g*<sub>*i*−1</sub>, где *c*<sub>*i*</sub> — это *i*-й
  символ алфавита
- ...

Заметим, что длины строки Грея очень быстро растут (длина *i*-й строки равна 2<sup>*i*</sup> − 1), но они имеют
регулярную фрактальную структуру.

Часто на строках Грея удобно разбирать работу различных строковых алгоритмов.

В этой задаче вам надо по заданному индексу *k* строки Грея и по числу *j* вывести *z*<sub>*j*</sub> — значение
z-функции строки *g*<sub>*k*</sub> в индексе *j*.

## Входные данные

В первой строке входных данных записано целое число *t* (1 ≤ *t* ≤ 10<sup>4</sup>) — количество наборов входных данных в
тесте.

Каждый набор входных данных состоит из строки, в которой записаны два целых числа *k* и *j* (1 ≤ *k* ≤ 26, 0 ≤ *j* <
2<sup>*k*</sup> − 1).

## Выходные данные

Выведите *t* целых чисел — ответы для наборов входных данных в тесте, то есть значение z-функции для *g*<sub>*k*</sub> в
индексе *j*.

## Пример

**Входные данные**

```
8
1 0
2 0
2 1
2 2
3 0
3 3
3 6
3 4
```

**Выходные данные**

```
0
0
0
1
0
0
1
3
```
